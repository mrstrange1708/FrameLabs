# Code of Conduct

## Our Pledge

We as members, contributors, and maintainers pledge to make participation in the FrameLabs project a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our Standards

### Expected Behaviour

- Use welcoming and inclusive language
- Be respectful of differing viewpoints and experiences
- Gracefully accept constructive criticism
- Focus on what is best for the project and the community
- Show empathy towards other contributors
- Write clear, well-documented code with meaningful commit messages
- Follow the established code style and conventions
- Review pull requests constructively and promptly

### Unacceptable Behaviour

- Trolling, insulting/derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing others' private information without explicit permission
- Pushing directly to protected branches without review
- Committing secrets, credentials, or environment files to the repository
- Intentionally introducing broken or untested code
- Ignoring CI/CD pipeline failures
- Bypassing code review processes

## Collaboration Standards

- All changes must go through pull requests — no direct pushes to `main`
- Pull requests require at least one approving review before merging
- Merge conflicts must be resolved before merging
- Keep pull requests focused and reasonably sized
- Write descriptive PR titles and descriptions

## Repository Hygiene

- Do not commit generated files, build artefacts, or `node_modules`
- Do not commit environment variable files (`.env`, `.env.local`, etc.)
- Keep dependencies up to date and review security advisories
- Delete branches after merging

## Enforcement

Instances of unacceptable behaviour may be reported to the project maintainers. All complaints will be reviewed and investigated, resulting in a response deemed necessary and appropriate to the circumstances.
